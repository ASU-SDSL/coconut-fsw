8/*
9* File Downlink Task
10* - This task is responsible for sending files to the ground on command
11* - This task uses a simple custom protocol based on go-back-n
12*
13* The protocol:
14* - The ground sends a "file downlink initialization" command
15* - The satellite responds with the first n packets of the file
16* - The ground sends an ack command after every n packet is received, specified by the sequence number (for the file, unrelated to ccsds space packet)
17* - The satellite will increment the window of packets it sends when acks are received in order
18* - If the satellite receives all acks up to the most recently sent sequence number within ACK_TIMEOUT, it immediately sends the next n packets
19* - If the satellite does not receive all acks within ACK_TIMEOUT, it will send the packets starting at the last in-order ack received sequence number
20*/
21
